# About OpenClaw Agent Control

OpenClaw Agent Control is an operations console for monitoring and controlling multi-agent runtime workflows.

It is designed for teams that need a practical bridge between:
- execution visibility (what is running, stalled, blocked, idle), and
- operational action (triage, restart, reassignment, and incident follow-up).

## Mission
Provide a clear, fast, and operator-friendly interface to reduce mean time to detect and resolve agent runtime issues.

## What makes this project different
- Risk-first UI: critical information appears before secondary analytics.
- Explicit status semantics: `idle`, `executing`, `waiting`, `stalled`, `blocked`.
- Skill-oriented deployment: one command to deploy/start through packaged skills.
- Lightweight stack: FastAPI + Next.js with script-based operations.
- Bilingual documentation: Chinese and English guides for onboarding.

## Audience
- Agent platform engineers
- AI workflow operators
- Internal DevOps/SRE teams integrating OpenClaw workflows
